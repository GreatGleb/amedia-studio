# Процесс разработки с ИИ (AI Workflow)

Этот документ описывает стандартный процесс работы над фичами и багами с использованием ИИ-ассистентов.

## Шаги разработки

### 1. Исследование и анализ (Research)
- ИИ изучает текущую кодовую базу.
- Обсуждение требований с пользователем.
- Проверка папки `ai/patterns/` на соответствие принципам.

### 2. Планирование (Planning)
- ИИ создает `implementation_plan.md` в папке `.gemini` или специальной директории (или прямо в `ai/plans/`).
- Пользователь утверждает план.
- Создается `task.md` для отслеживания прогресса.

### 3. Реализация (Execution)
- ИИ выполняет задачи по одной, отмечая прогресс в `task.md`.
- Каждый шаг проверяется на работоспособность.

### 4. Проверка и полировка (Verification)
- Запуск тестов (если есть).
- Проверка UI на соответствие "премиальному" дизайну (плавность анимаций, отсутствие скачков).
- Создание `walkthrough.md` с отчетом.

## Правила работы с Git
- Создавать отдельную ветку для каждой фичи (если применимо).
- Делать атомарные коммиты с понятными сообщениями.
