# Принципы разработки (Patterns & Principles)

В этом файле собраны ключевые принципы, которым должен следовать код в этом проекте.

## KISS (Keep It Simple, Stupid)
- Не усложняйте код без необходимости.
- Если задачу можно решить простым CSS, не используйте JS.
- Разбивайте сложные компоненты на более мелкие и понятные.

## SOLID
- **S: Single Responsibility** — Каждый компонент или функция должны отвечать только за одну вещь.
- **O: Open/Closed** — Код должен быть открыт для расширения, но закрыт для модификации.
- **L: Liskov Substitution** — (Менее применимо в React, но важно для классов/типов).
- **I: Interface Segregation** — Не заставляйте компоненты зависеть от пропсов, которые они не используют.
- **D: Dependency Inversion** — Зависьте от абстракций, а не от конкретных реализаций.

## DDD (Domain-Driven Design) в Frontend
- Группируйте код по бизнес-фичам (доменам), а не по техническим ролям (например, папка `portfolio/`, а не `components/` для всего подряд).
- Выделяйте "чистую" бизнес-логику из React-компонентов в отдельные функции/хуки.

## Best Practices для нашего стека
- **React 19:** Используйте новые хуки (например, `use`) и экшены (Actions), где это уместно. Избегайте устаревших паттернов.
- **Tailwind CSS 4:** Используйте новые возможности v4 (например, улучшенные переменные, новые утилиты).
- **GSAP:** Всегда очищайте анимации при размонтировании компонентов (используйте `gsap.context()` или хуки очистки).
